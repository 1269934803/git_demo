  • git init 初始化仓库
  • git status 查看文件状态
  • git add 加文件名 追踪文件 也就是将文件放到暂存区中
  • git commit -m 提交信息（说明） 将文件提交到仓库中
  • git log 查看提交记录
  • 用暂存区中的文件覆盖当前工作目录中的问价 git checkout 文件
  • git rm --cached test.html 将文件从暂存区中删除
  • git reset --hard commitID 将git仓库中指定的更新记录恢复并覆盖暂存区和工作目录（也就是说所有的状态都会回到恢复的这次提交时，在这次提交之后的暂存区、工作目录或是提交记录都不存在了）
  • 分支与分支之间在开发过程中是独立的
  • 在切换分支之前一定要将分支暂存区中的文件提交到仓库中
  • git branch 查看分支
  • git branch 分支名称 创建分支
  • git checkout 分支名称 切换分支
  • git merge 被合并分支的名称
  • git branch -d 分支名称 删除分支 如果要删除的分支没有合并但是还想删除（git branch -D 分支名称 强制删除）
  • git stash 存储临时改动 当临时切换分支时，不想将当前分支的文件提交到仓库，可以用此命令将文件暂时保存更改，然后切换到其它分支工作
  • git stash pop 恢复存储临时改动
  • 向远程github仓库推送文件 git push 仓库地址 分支名称
  • 给远程仓库地址起别名 git remote add origin 仓库地址
  • 有别名之后 就可以用别名推送文件了 git push origin master
  • git push -u origin master (-u 参数会记住推送地址和分支名 下次提交时可以直接使用git push 命令)
